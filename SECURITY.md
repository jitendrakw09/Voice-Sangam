# 🔒 Security Policy

## 📌 Supported Versions
The following versions of this project are currently being supported with security updates:

| Version | Supported          |
|---------|------------------|
| Latest  | ✅ Yes            |
| Older   | ❌ No             |

---

## 🚨 Reporting a Vulnerability
We take security issues seriously and appreciate responsible disclosure.

If you discover a vulnerability, please follow these steps:

- **Do not** open a public issue.
- Email the maintainers directly at: **your-email@example.com**
- Provide a detailed description of the vulnerability, including:
  - Steps to reproduce
  - Potential impact
  - Suggested fix (if available)

---

## ⏱ Response Time
We aim to respond to security reports within:

- **Acknowledgement:** Within 48 hours  
- **Initial Assessment:** Within 3–5 business days  
- **Resolution Timeline:** Depends on severity and complexity  

---

## 🛡 Security Best Practices
To ensure the security of this project:

- Keep dependencies up to date  
- Avoid exposing sensitive information (API keys, credentials)  
- Use environment variables for configuration  
- Follow secure coding practices  
- Validate and sanitize all user inputs  

---

## 🔐 Disclosure Policy
- Vulnerabilities will be disclosed responsibly after a fix is released  
- Credit will be given to the reporter (if desired)  

---

## ⚠️ Scope
This policy applies to:

- Source code in this repository  
- Deployment configurations  
- API endpoints (if applicable)

We appreciate the efforts of security researchers and contributors who help improve this project.
